<task type="auto">
  <name>Task 4: Implement composite guardrail</name>
  <files>src/security/composite-guardrail.ts</files>
  <action>
Create src/security/composite-guardrail.ts:

1. CompositeGuardrail<TInput, TOutput> extends BaseGuardrail
   - name = 'CompositeGuardrail'
   - priority = 0
   - description = 'Orchestrates multiple guardrails'

2. Constructor:
   - guardrails: BaseGuardrail<any, any>[]
   - mode: 'all' | 'any' = 'all' (all must pass vs any can block)

3. In constructor, sort guardrails by priority (ascending)

4. validate() method:
   - Iterate through guardrails in priority order
   - For each: call guardrail.execute(input, context)
   - Collect all results
   - If mode='all' and any blocked: return first block result
   - If mode='any' and any blocked: return combined result
   - Return final output (chained through guardrails)
   - Include all guardrailResults in metadata

5. Helper to determine highest severity from results

Reference Pattern 2 composite example from 01-RESEARCH.md.
